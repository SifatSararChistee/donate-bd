1. What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents, particularly HTML and XML. It represents the structure of a document as a tree of objects, with each element, attribute, and piece of text in the document represented as a node in the tree. Through the DOM, programs (such as JavaScript) can interact with the document, allowing for dynamic manipulation of the content, structure, and style of a webpage.

2. How do you select an element from the DOM?

To select an element from the DOM in JavaScript, you can use several methods depending on how you want to target the element (by id, class, tag, or using CSS selectors). Below are the most common ways to select elements:

1. Selecting by id: You can use the document.getElementById() method to select an element by its unique id.

2. Selecting by class: To select elements by their class name, use document.getElementsByClassName(). This method returns a live HTMLCollection (an array-like object) of all elements with the specified class.

3. Selecting by tag: You can select all elements of a specific tag using document.getElementsByTagName(). It returns an HTMLCollection of elements with the specified tag name.

4. Selecting using CSS selectors: The document.querySelector() method returns the first element that matches a CSS selector, while document.querySelectorAll() returns all matching elements as a static NodeList (an array-like object).

5. Selecting by name attribute: For form elements, you can use document.getElementsByName() to select elements by their name attribute.

3. What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in JavaScript where you take advantage of event bubbling to manage events more efficiently. Instead of attaching event listeners to multiple child elements, you attach a single event listener to a parent element. This event listener can then respond to events triggered by any of its child elements.
